Dokumentasi API Pandu Backend

Fokus backend: Auth + Kode Registrasi + Modul + Sesi + Quiz + Enrollment (berdasarkan peran)
BASIC URL
Peran: admin, teacher, student
API create module: POST /api/modules (teacher/admin)
Lokasi file konten: storage/modules/{moduleId}/sessions/{sessionId}
Lokasi file banner: storage/modules/{moduleId}/banner

Endpoint Autentikasi

Registrasi

POST/api/auth/register
{
  "name": "Siswa 1",
  "email": "student1@mail.com",
  "password": "password123",
  "registration_code": "938214"
}
Register menerima application/json dan text/plain (isi wajib JSON valid).
Role user saat register ditentukan otomatis dari target role kode registrasi.

Masuk

POST/api/auth/login
{
  "email": "teacher1@mail.com",
  "password": "password123"
}
Login menerima application/json dan text/plain (isi wajib JSON valid).

Contoh Respons Login

{
  "success": true,
  "message": "Login berhasil",
  "data": {
    "user": {
      "id": 3,
      "name": "Guru 1",
      "email": "teacher1@mail.com",
      "role": "teacher"
    },
    "token": "JWT_TOKEN",
    "token_type": "Bearer"
  }
}

Logout

POST/api/auth/logout
curl -X POST "{{BASE_URL}}/api/auth/logout" \
  -H "Authorization: Bearer JWT_TOKEN"
Setelah logout, token langsung masuk blacklist dan tidak bisa dipakai lagi.

Endpoint Kode Registrasi

Admin/Teacher dapat membuat kode registrasi 4-6 digit
Teacher hanya bisa membuat kode untuk role student
Admin bisa membuat kode untuk role student atau teacher
POST /api/auth/register-codes                 (admin/teacher)
GET /api/auth/register-codes                  (admin/teacher)
GET /api/auth/register-codes/summary          (admin/teacher)
GET /api/auth/register-codes/:codeId/usages   (admin/teacher)
PATCH /api/auth/register-codes/:codeId/revoke (admin/teacher)
PATCH /api/auth/register-codes/expired/archive (admin/teacher)
DELETE /api/auth/register-codes/expired       (admin/teacher)
Endpoint list kode/usages sudah mendukung metadata list standar: meta.page, meta.per_page, meta.total, meta.total_pages.
Setiap item list kode menyertakan capabilities agar frontend tidak menebak izin aksi.

Body Create Kode Registrasi

{
  "target_role": "student",
  "code_length": 6,
  "max_uses": 10,
  "expires_in_hours": 24
}

Contoh Create Kode (teacher)

curl -X POST "{{BASE_URL}}/api/auth/register-codes" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -H "Content-Type: application/json" \
  -d '{
    "target_role": "student",
    "code_length": 6,
    "max_uses": 20,
    "expires_in_hours": 48
  }'

Contoh Daftar Kode & Riwayat Penggunaan

curl -X GET "{{BASE_URL}}/api/auth/register-codes" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER"

curl -X GET "{{BASE_URL}}/api/auth/register-codes/12/usages" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER"

Contoh Summary Kode Registrasi

curl -X GET "{{BASE_URL}}/api/auth/register-codes/summary" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER"

# contoh response
{
  "success": true,
  "data": {
    "total_codes": 12,
    "active_codes": 7,

Contoh Nonaktifkan Kode

Endpoint Modul

Teacher/Admin: membuat, mengubah, menghapus modul dan konten sesi
Student: membaca modul yang sudah di-enroll
GET /api/modules                     (semua yang diizinkan berdasarkan peran)
    GET /api/modules/:moduleId           (harus punya akses)
    GET /api/modules/:moduleId/banner    (harus punya akses)
    POST /api/modules                    (teacher/admin, multipart: name, description?, banner?)
    PUT /api/modules/:moduleId           (teacher/admin pemilik modul)
    DELETE /api/modules/:moduleId        (teacher/admin pemilik modul)
    PATCH /api/modules/:moduleId/regenerate-enroll-key (teacher/admin pemilik modul)
List modul (GET /api/modules) memakai shape standar list: { success, data, meta }.
Setiap modul menyertakan capabilities (mis: can_edit, can_delete, can_manage_sessions).

Field Multipart Buat Modul

name: string (wajib)
    description: string (opsional)
    banner: file (opsional, pdf/gambar/video sesuai filter backend)

    Perilaku otomatis:
    - membuat enroll_key (kunci bersama per modul)
    - otomatis membuat 3 sesi awal: Sesi 1, Sesi 2, Sesi 3

Field Multipart Update Modul

name: string (opsional)
    description: string (opsional, kirim kosong untuk null)
    banner: file (opsional)
    enroll_key: string (opsional, unik antar module)

    Validasi enroll_key saat update:
    - hanya huruf, angka, dan tanda minus (-)
    - maksimal 64 karakter
    - jika sudah dipakai module lain, update ditolak

Contoh Request Create Module (tanpa banner)

curl -X POST "{{BASE_URL}}/api/modules" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "name=Modul Algebra Dasar" \
  -F "description=Materi aljabar untuk semester 1"

Contoh Request Create Module (dengan banner)

curl -X POST "{{BASE_URL}}/api/modules" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "name=Modul Trigonometri" \
  -F "description=Pengantar trigonometri" \
  -F "banner=@C:/path/banner.jpg"

Contoh Respons Create Module

{
  "success": true,
  "message": "Module berhasil dibuat dengan 3 sesi default",
  "data": {
    "id": 12,
    "teacher_id": 3,

    "banner_image_path": "storage/modules/12/banner/17100000-banner.jpg",
    "banner_download_url": "/api/modules/12/banner",
    "enroll_key": "A1B2C3D4E5F6",
    "created_at": "2026-04-17T10:00:00.000Z"
  }
}

Contoh Ambil Banner Modul

curl -X GET "{{BASE_URL}}/api/modules/12/banner" \
  -H "Authorization: Bearer JWT_TOKEN_STUDENT"

Contoh Update, Delete, dan Regenerate Key

curl -X PUT "{{BASE_URL}}/api/modules/12" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "name=Modul Trigonometri Lanjut" \
  -F "description=Update deskripsi" \
  -F "enroll_key=TRIGONOMETRI-KELAS-7A"

  -H "Authorization: Bearer JWT_TOKEN_TEACHER"

curl -X DELETE "{{BASE_URL}}/api/modules/12" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER"
Jika enroll_key sudah digunakan module lain, API akan menolak dengan pesan: "Enroll key tidak bisa digunakan, coba gunakan nama lain".

Contoh Error Update Enroll Key

# 400 - format invalid
{
  "success": false,
  "message": "Format enroll_key tidak valid. Gunakan hanya huruf, angka, dan tanda minus (-)"
}

# 409 - duplikat enroll_key
{
  "success": false,
  "message": "Enroll key tidak bisa digunakan, coba gunakan nama lain"
}

Endpoint Sesi

GET /api/modules/:moduleId/sessions
POST /api/modules/:moduleId/sessions                 (teacher/admin)
PUT /api/modules/:moduleId/sessions/:sessionId       (teacher/admin)
DELETE /api/modules/:moduleId/sessions/:sessionId    (teacher/admin)
GET /api/modules/:moduleId/sessions/:sessionId/schedule
PATCH /api/modules/:moduleId/sessions/:sessionId/schedule (teacher/admin)
GET /api/modules/:moduleId/sessions/:sessionId/reminder
PUT /api/modules/:moduleId/sessions/:sessionId/reminder
DELETE /api/modules/:moduleId/sessions/:sessionId/reminder

GET /api/modules/:moduleId/sessions/:sessionId/contents
GET /api/modules/:moduleId/sessions/:sessionId/contents/:contentId/file
POST /api/modules/:moduleId/sessions/:sessionId/contents/:contentId/view-url
POST /api/modules/:moduleId/sessions/:sessionId/contents        (teacher/admin)
PUT /api/modules/:moduleId/sessions/:sessionId/contents/:contentId (teacher/admin)
DELETE /api/modules/:moduleId/sessions/:sessionId/contents/:contentId (teacher/admin)
Jadwal sesi default kosong (open_at = null) saat module/sesi dibuat.
Jika open_at di masa depan, student tidak bisa membaca konten sesi/file sampai waktu tersebut.
Untuk mendapatkan contentId, ambil dari endpoint GET /api/modules/:moduleId/sessions/:sessionId/contents pada field id.
Status endpoint delete sesi: tersedia di DELETE /api/modules/:moduleId/sessions/:sessionId.
List sesi & konten juga menyertakan meta pagination dan capabilities per item.

Payload Tambah Konten Sesi

content_type: file | url | text (wajib)
title: string (opsional)
file: binary (wajib jika content_type=file)
url: string (wajib jika content_type=url)
text_content: string (wajib jika content_type=text, opsional untuk file/url)

Field Tambahan di Response Konten

file_download_url: string | null
file_kind: image | video | document | other | null
is_media: boolean
is_public_view_supported: boolean
should_use_public_view_url: boolean

Catatan frontend:
- Jika is_public_view_supported=true, frontend bisa generate public view URL
- Jika is_media=true (image/video), biasanya tidak perlu public view URL

Contoh Create Sesi Baru

curl -X POST "{{BASE_URL}}/api/modules/12/sessions" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Sesi Turunan"
  }'

Contoh Update Sesi

curl -X PUT "{{BASE_URL}}/api/modules/12/sessions/33" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Sesi Turunan Dasar",
    "sort_order": 2
  }'
Validasi: sort_order harus angka bulat lebih dari 0.

Contoh Delete Sesi

curl -X DELETE "{{BASE_URL}}/api/modules/12/sessions/33" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER"

Contoh Atur Jadwal Sesi

Endpoint Quiz Per Sesi

Struktur: module -> session -> quiz (1 quiz per sesi)
Tipe soal didukung: mcq dan essay
MCQ dinilai otomatis sistem, Essay dinilai manual oleh teacher/admin.
GET /api/modules/:moduleId/sessions/:sessionId/quiz
POST /api/modules/:moduleId/sessions/:sessionId/quiz                    (teacher/admin, multipart)
PUT /api/modules/:moduleId/sessions/:sessionId/quiz                     (teacher/admin, multipart)
PATCH /api/modules/:moduleId/sessions/:sessionId/quiz/publish           (teacher/admin)
GET /api/modules/:moduleId/sessions/:sessionId/quiz/banner

POST /api/modules/:moduleId/sessions/:sessionId/quiz/questions           (teacher/admin, multipart)
PUT /api/modules/:moduleId/sessions/:sessionId/quiz/questions/:questionId (teacher/admin, multipart)
DELETE /api/modules/:moduleId/sessions/:sessionId/quiz/questions/:questionId (teacher/admin)
GET /api/modules/:moduleId/sessions/:sessionId/quiz/questions/:questionId/media

POST /api/modules/:moduleId/sessions/:sessionId/quiz/start               (student)
POST /api/modules/:moduleId/sessions/:sessionId/quiz/submit              (student)

GET /api/modules/:moduleId/sessions/:sessionId/quiz/attempts             (teacher/admin)
GET /api/modules/:moduleId/sessions/:sessionId/quiz/attempts/:attemptId  (teacher/admin)
PATCH /api/modules/:moduleId/sessions/:sessionId/quiz/attempts/:attemptId/review (teacher/admin)

Payload Create/Update Quiz (multipart)

title: string (wajib saat create)
description: string (opsional)
banner: file (opsional, image/video/pdf sesuai filter backend)
remove_banner: boolean (opsional, untuk hapus banner saat update)
duration_minutes: integer 1..480
max_attempts: integer 1..50
passing_score: number 0..100

Contoh Create Quiz

curl -X POST "{{BASE_URL}}/api/modules/12/sessions/33/quiz" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "title=Quiz Sesi Turunan" \
  -F "description=Evaluasi pemahaman turunan dasar" \
  -F "duration_minutes=30" \
  -F "max_attempts=2" \
  -F "passing_score=75" \
  -F "banner=@C:/path/quiz-banner.jpg"

Contoh Tambah Soal MCQ (dengan media opsional)

curl -X POST "{{BASE_URL}}/api/modules/12/sessions/33/quiz/questions" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "question_type=mcq" \
  -F "question_text=Hasil turunan x^2 adalah..." \
  -F "points=10" \
  -F "options=[{\"option_text\":\"2x\",\"is_correct\":true},{\"option_text\":\"x\",\"is_correct\":false}]" \
  -F "media=@C:/path/soal-1.png"

Contoh Tambah Soal Essay

curl -X POST "{{BASE_URL}}/api/modules/12/sessions/33/quiz/questions" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \
  -F "question_type=essay" \
  -F "question_text=Jelaskan konsep turunan dalam 3-5 kalimat" \

Contoh Publish Quiz

curl -X PATCH "{{BASE_URL}}/api/modules/12/sessions/33/quiz/publish" \
  -H "Authorization: Bearer JWT_TOKEN_TEACHER" \

Alur Cepat End-to-End

1) Admin/Teacher login -> dapat token
2) Admin/Teacher create registration code (4-6 digit)
3) User register menggunakan registration_code -> role mengikuti target_role kode
4) Teacher create module -> dapat enroll_key + 3 sesi default
5) Teacher (opsional) atur jadwal buka sesi via open_at
6) Teacher isi konten sesi (url/text/file)
7) Teacher buat quiz per sesi + tambah soal MCQ/Essay + publish
8) Student enroll pakai enroll_key
9) Student baca module, kerjakan quiz sesi (start -> submit)
10) Sistem auto-nilai MCQ, Teacher/Admin review Essay
11) Jika perlu, user logout agar token saat ini tidak bisa dipakai lagi
